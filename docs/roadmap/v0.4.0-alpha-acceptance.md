## Validation Commands

The validation set remains:

```bash
cargo fmt --all
cargo clippy -- -D warnings
cargo test -q compile_only_samples_tests -- --nocapture
cargo test -q run_headless_scripts -- --nocapture
cargo test -q samples_have_no_legacy_keys -- --nocapture
cargo test -q --test listener_twin_validation_fixture -- --nocapture
zola build
mdbook build docs/rhai_book
RUST_BACKTRACE=1 cargo test -- --nocapture
```

When `src/` changes are included, keep the mandatory reports current:

```bash
set -o pipefail
( RUST_BACKTRACE=1 cargo test -- --nocapture ) 2>&1 | tee test_report.txt
status=$?
echo "cargo test exit=$status @ $(date -Iseconds)" > test_status.txt
exit $status
```

## Remaining Release Decision

The repository is not ready for a v0.4.0 alpha release candidate. The release
gate has been reset around rhythm/harmony ecology, and the listener twin / DCC
coupling work is now in scope. Before changing `Cargo.toml` from `0.4.0-dev` to
the final release version, complete the rhythm redesign plan, rebuild the
curated demo set, and pass the new audition and reporting gates.

The listener twin / DCC work (criteria 13-17) is now complete: the engine is
implemented and covered by tests, and the composer documentation has landed in
the Rhai Book. Phase 5 (listener memory and expectation) stays out of v0.4.0
scope. The remaining release blockers are the rhythm/harmony audition decisions
(criteria 2, 3, 4, 6) and the website roadmap scope update (criterion 12).
