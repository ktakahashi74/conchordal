```bash
mdbook build docs/rhai_book

```

When `src/` changes are included, keep the mandatory reports current:

```bash
set -o pipefail
( RUST_BACKTRACE=1 cargo test -- --nocapture ) 2>&1 | tee test_report.txt
status=$?
echo "cargo test exit=$status @ $(date -Iseconds)" > test_status.txt
exit $status
```

## Remaining Release Decision

All acceptance criteria pass. The rhythm redesign landed (criteria 2, 4, 6),
the listener twin / DCC work is complete (criteria 13-17), and the scripting
surface was consolidated after the checklist reset: `pitch_mode`/boilerplate
research controls were promoted or absorbed into core verbs (`anchor()`,
phonation-resolved pitch apply), the API was layered into
Core/Experimental/Tuning/Research tiers, and the Rhai Book reference is now
generated from the engine with CI drift tests, so criteria 6/7 hold
structurally. Criterion 12 closed with the README/site updates. Criterion 3
was re-scoped to functional integration after the 2026-06-11 flagship
audition. On 2026-06-12 the samples were rebuilt from zero as a twelve-étude
book (`samples/01_a_single_voice.rhai` .. `12_emergence_and_resolution.rhai`,
with assays under `samples/research/`), so the first-touch alpha path walks
the instrument concept by concept. Phase 5 (listener memory and expectation)
stays out of v0.4.0 scope.

The remaining step is mechanical: change `Cargo.toml` from `0.4.0-dev` to the
release version and tag.

Carried explicitly to the beta: the musically convincing showcase. The arc
étude's audition failed across pulse, harmonic arch, and form while all
functional metrics pass — the gap between verified mechanisms and music is
the beta's central problem statement, not a v0.4.0 parameter-tuning task.
