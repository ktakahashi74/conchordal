When `src/` changes are included, keep the mandatory reports current:

```bash
set -o pipefail
( RUST_BACKTRACE=1 cargo test -- --nocapture ) 2>&1 | tee test_report.txt
status=$?
echo "cargo test exit=$status @ $(date -Iseconds)" > test_status.txt
exit $status
```

## Remaining Release Decision

The repository is not yet a v0.4.0 alpha release candidate, but the gate has
narrowed. The rhythm redesign landed: the three families are perceptually
separated on a coupled-oscillator meter core, the rhythm API is exposed as
composer-facing concepts on both species and groups, and the curated demos,
reports, and Rhai Book docs are in place (criteria 2, 4, 6).

The listener twin / DCC work (criteria 13-17) is complete: the engine is
implemented and covered by tests, and the composer documentation has landed in
the Rhai Book. Phase 5 (listener memory and expectation) stays out of v0.4.0
scope.

Two release blockers remain before changing `Cargo.toml` from `0.4.0-dev` to the
final release version:

- Criterion 3: a musically convincing flagship showcase. `conchordal_ecology.rhai`
  is the candidate but is not yet at the bar; a dedicated etude may be needed.
- Criterion 12: the website roadmap still needs a web-scope update.

Rhythm tuning beyond the initial separation audition — meter correspondence,
musicality, and tracking new API features — continues but is not a release
blocker.
