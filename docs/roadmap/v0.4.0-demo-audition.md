## Criteria

For each curated demo, decide:

- `keep`: suitable for the v0.4 alpha path as-is
- `revise`: keep the role, but tune duration, density, transitions, or levels

- whether the v0.4 concept is audible without reading implementation notes

## Audition Round 2026-06-03

Cross-family finding: metric beat, entrained beat, and flow timing all sound
like the same wobbling beat. The intended contrast (isochronous pulse vs.
emergent synchronization vs. non-metric flow) is not audible.

Confirmed root cause (code trace): every family builds its onset grid from a
single shared clock. `phonation_config_from_spec` gives metric and entrained
`PhonationClockConfig::ThetaGate`, and flow `Composite { subdivision: [5,7,11] }`
over the same theta gate. `ThetaGateClock` places every gate boundary at theta
phase crossings via `gate_clock::next_gate_tick(ctx.rhythms.theta, ...)`. That
theta is not a fixed grid: it is the `AdaptiveRhythm` oscillator in
`core/modulation.rs` (range 3-12 Hz, init 6 Hz) whose frequency `omega` and
phase `phi` continuously track the habitat audio. The drive is
`u_theta = tanh(500 * positive_spectral_flux)` from `core/stream/dorsal.rs`.

Consequences:

- The beat grid is a closed feedback loop: voices emit onsets -> habitat flux ->
  adaptive theta re-estimates omega/phi -> theta sets the gate grid -> voices
  emit onsets. As density and texture shift, theta drifts inside 3-12 Hz, which
  is heard as the shared wobble.
- A voice's `metric_beat(2.0)` rate is an accumulator rate counted in theta
  gates, not wall-clock Hz. "2 Hz" means roughly "fire every Nth wobbling theta
  gate", so the metric surface cannot be isochronous in real time while it rides
  the adaptive theta.
- The only engine-level differences (metric accent depth, entrained
  social/vitality coupling plus attack reward, flow renewal IOIs) are too subtle
  to survive the shared theta substrate, the shared `seek_consonance()` +
  `pitch_apply_mode("gate_snap")` pitch behavior, and the dominant sustained
  drone, so the families collapse perceptually.

Observability finding: it is hard to tell from the current GUI whether voices
are entraining to the neural rhythm. The Listener Twin mandala draws the
delta/theta clock hand and the Kuramoto order `r` is shown as a small number,
but individual voice onset phases are not overlaid on the rhythm circle, so
phase clustering (locked) vs. spread (unlocked) cannot be seen directly.

These are the current blockers for criteria 2, 3, 4, and 6.

### Engine fix landed 2026-06-03

A new `PhonationClockConfig::FixedRate { rate_hz }` clock was added so families
no longer all ride the adaptive theta:

- Metric beat uses a fixed wall-clock grid (subdivided 4x) anchored to absolute
  tick 0 (`shared_grid: true`) instead of `ThetaGate`, so the pulse is
  isochronous, does not drift with the audio, and is shared across voices. The
  onset rate matches the grid with a 3-gate refractory, so every metric voice
  fires on the beat together (a crisp chord) instead of at random offsets.
  Headless: downbeat group `ioi_cv` 0.007 (near-perfect).
- Flow timing fires its real-time renewal IOIs on a fixed fine grid (48 Hz,
  ~21 ms) with a **per-voice anchor** (`shared_grid: false`): droplets keep
  independent phase. A shared absolute grid made the droplets read as locked to
  one period; per-voice anchoring restores independent rain. Holds are scaled 3x
  (between the too-long legato and too-short click extremes).

Neural-rhythm entrainment to the metric beat: the onset-IOI lock now routes the
detected beat to the band that contains it. A metric beat (~2 Hz) is delta-rate,
below theta's 3 Hz floor, so it locks the **delta** oscillator (phase + omega)
instead of being clamped/lost in theta. Before this, delta sat at its ~1 Hz init
and theta wandered 3-5 Hz: the neural rhythm did not entrain the beat at all.
After: delta tracks the beat region (~2.0-2.8 Hz in the metric demo). It is not a
pristine 2.0 lock because the demo mixes 2 Hz and 3 Hz layers and the onset
tracker uses a single pooled IOI; clean per-tempo locking would need beat
induction (a deeper feature, deferred). The metric generation clock stays a fixed
external pulse — only the listener-side neural rhythm follows it.

Pitch articulation (`gate_snap`) is now snap-and-hold, matching the family
concepts (metric = stable-pitch beats; flow = discrete-pitch droplets):

- A `gate_snap` voice adopts the consonance target at each note onset and holds
  that pitch for the note; successive notes step toward consonance discretely.
  Previously the small-drift branch applied the moving hill-climb target every
  control tick, which—now that notes are longer—was audible as a glissando.
  Large-jump gate ducking (for sustained retunes) is unchanged.
- Metric `offbeat` now holds two grid gates (`cycles(2)`) so the syncopation
  reads as a note rather than a fine tick.

Two further theta-decoupling fixes apply to both fixed-rate families:

- Onset firing uses a flat timing field (weight 1.0), not the adaptive
  `env_open`. Fixed-rate clocks are external/renewal clocks; gating them by the
  theta-periodic envelope would re-couple them to theta (metric loses its steady
  pulse, flow droplets lock to the theta period).
- Note-off scheduling extends the grid at the clock's own fixed period, not via
  the adaptive theta. Previously a fixed-rate note whose hold reached past the
  current hop got its note-off placed on a theta-spaced grid, so note length
  drifted with theta and articulation (legato / staccato / marcato) varied by
  onset timing. Now note length is constant per layer.
- Entrained beat keeps `ThetaGate` by design: it should track the emergent
  population rhythm, so its drift is now meaningful contrast rather than the
  shared wobble.

The dead `Composite` / subdivision / internal-phase clock machinery (its only
user was the old flow path) was removed.

Headless guardrail after the fix (per pulse group):

| Family | beat_stability | burstiness |
|---|---:|---:|
| Metric (grp 2) | 0.89 | -0.81 |
| Entrained (grp 2) | 0.00 | +0.18 |
| Flow (grp 2) | 0.11 | -0.06 |

The metric/entrained contrast is now strong at the metric level (0.89 vs 0.00).
Open tuning item for audition: flow burstiness fell to ~0 (was positive), so the
fixed grid may have over-regularized the rain-like clustering; revisit flow
cluster parameters if it still reads as too even.

Per-voice phase-driven entrained onsets (making the lock-in process directly
audible, not only emergent-theta drift) are deferred; in the meantime voice
phase coherence is overlaid directly on the Listener Twin mandala (violet dots
on the theta ring plus a Kuramoto mean-resultant vector), so entrainment reads
on the same compass as the listener rhythm.

Known limitation from the code review: `beat_strength`/`accent` has no engine
effect on the one-onset-per-beat metric grid (a within-gate cosine only shapes
sub-beat candidates, which metric does not emit). A meter-level accent (emphasis
every Nth beat) is the right home for it and is not yet implemented; the inert
sub-theta mapping was removed so it is no longer misleading.

## Current Rhythm Candidate Notes

### entrained_beat.rhai

- First-pass issue: attack felt too strong and the sound decayed too quickly.
- Revision: reduced level slightly, lengthened gate duration, slowed attack,
  lengthened decay/release, and kept the entrainment role intact.
- Decision: revise, re-audition.

### flow_timing_field.rhai

- First-pass issue: flow timing sounded almost metric.
- First-pass issue: attack felt too strong and the sound decayed too quickly.
- Second-pass issue: note length varied too much by onset timing, with some
  notes decaying immediately and others sustaining much longer.
- Third-pass issue: direct pink IOI made the whole rhythm feel like a slow
  tempo drift, not rain drops or river flow.
- Revision: changed flow timing to a stationary renewal process with local
  jitter and short rain-like clusters. Dense non-binary clock points now only
  provide timing resolution; reduced onset kick strength; lowered rates;
  lengthened gates and ADSR.
- Revision: changed fixed-gate duration semantics so `gates(n)` holds for `n`
  gates from the actual onset phase, not from the current gate boundary.
- Decision: revise, re-audition.

### rhythm_harmony_ecology.rhai

- Role: functional integration demo, not the musical showcase.
- First-pass issue: shared field-driven timing made the whole rhythm slowly
  wobble, and the result sounded more like a feature stack than a piece.
- Revision: retuned as a stable-groove probe with metric beat, entrained agents,
  and a lower-level non-metric flow layer.
- Decision: re-audition as a feature probe. Do not use it to judge whether
  v0.4.0 has a musically convincing flagship.

### conchordal_ecology.rhai

- Revision: applied the softer entrained and flow voicing changes to the
  integration candidate so it does not inherit the plucked attack problem.
- Revision: inherits the corrected fixed-gate duration semantics for flow
  particles.
- Decision: candidate material for the separate musical showcase, but it may
  need to become an etude rather than a feature-complete ecology demo.

## Demoted Demo Notes

### consonance_field_control.rhai

- Audible concept: first audition found the concept too slow and scattered; second pass was still too scattered.
- Duration/density: revised toward audible field-state stanzas followed by one guide voice instead of a density group.
- Level/transition issues: audible anchor was irritating; revised anchor to `send(habitat_bus)`.
- Movement/loudness: glide itself does not gate-fade, but entrain vitality can reduce target amplitude while the voice crosses low-consonance regions. Revised the guide to `send(presentation_bus)` so movement does not also audition viability.
- Articulation: `gate_snap` was more Western-music-like and easier to parse than glide, so it is now the audition candidate.
- Demo clarity: prior versions did not make the field-control function audible. Revised to sound each field state first, then let the same guide snap to the new state.
- Decision: demote from the curated v0.4 alpha path. Keep as a research comparison unless it later becomes musically legible without explanation.
