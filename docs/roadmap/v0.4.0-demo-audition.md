# Conchordal v0.4.0 Demo Audition

Status: Rhythm candidates under audition
Updated: 2026-05-08

## Criteria

For each curated demo, decide:

- `keep`: suitable for the v0.4 alpha path as-is
- `revise`: keep the role, but tune duration, density, transitions, or levels
- `demote`: move out of the curated path and treat as research comparison

Listen for:

- whether the v0.4 concept is audible without reading implementation notes
- whether the script is short enough for a release-entry demo
- whether density, loudness, and transitions are stable
- whether the demo name matches what the listener hears

## Rhythm Redesign Gate

The previous curated audition path is paused. v0.4.0 now requires rhythm as a
core part of the musical ecology, not only as `pulse_foundation.rhai`.

New required audition families:

| Family | Intended role | Target duration | Audition status | Decision |
|---|---|---:|---|---|
| Metric beat | Western-music-like pulse surface | 30-60s | Candidate built | Pending first pass |
| Entrained beat | Synchronization emerging from agent coupling and survival | 30-60s | First pass heard too plucked | Revised, re-audition |
| Flow timing | Rain/river-like non-metric onset texture | 30-60s | Earlier passes sounded too metric, too plucked, then too slowly drifting | Revised, re-audition |
| Integrated ecology | Consonance, rhythm, viability, movement, and respawn in one flagship | 45-90s | Revised after entrained/flow feedback | Re-audition |

See `docs/roadmap/v0.4.0-rhythm-redesign.md`.

## Current Rhythm Candidate Notes

### entrained_beat.rhai

- First-pass issue: attack felt too strong and the sound decayed too quickly.
- Revision: reduced level slightly, lengthened gate duration, slowed attack,
  lengthened decay/release, and kept the entrainment role intact.
- Decision: revise, re-audition.

### flow_timing_field.rhai

- First-pass issue: flow timing sounded almost metric.
- First-pass issue: attack felt too strong and the sound decayed too quickly.
- Second-pass issue: note length varied too much by onset timing, with some
  notes decaying immediately and others sustaining much longer.
- Third-pass issue: direct pink IOI made the whole rhythm feel like a slow
  tempo drift, not rain drops or river flow.
- Revision: changed flow timing to a stationary renewal process with local
  jitter and short rain-like clusters. Dense non-binary clock points now only
  provide timing resolution; reduced onset kick strength; lowered rates;
  lengthened gates and ADSR.
- Revision: changed fixed-gate duration semantics so `gates(n)` holds for `n`
  gates from the actual onset phase, not from the current gate boundary.
- Decision: revise, re-audition.

### conchordal_ecology.rhai

- Revision: applied the softer entrained and flow voicing changes to the
  integrated flagship so it does not inherit the plucked attack problem.
- Revision: inherits the corrected fixed-gate duration semantics for flow
  particles.
- Decision: re-audition after the single-family demos.

## Demoted Demo Notes

### consonance_field_control.rhai

- Audible concept: first audition found the concept too slow and scattered; second pass was still too scattered.
- Duration/density: revised toward audible field-state stanzas followed by one guide voice instead of a density group.
- Level/transition issues: audible anchor was irritating; revised anchor to be field-only.
- Movement/loudness: glide itself does not gate-fade, but entrain vitality can reduce target amplitude while the voice crosses low-consonance regions. Revised the guide to `brain("drone").presentation_only()` so movement does not also audition viability.
- Articulation: `gate_snap` was more Western-music-like and easier to parse than glide, so it is now the audition candidate.
- Demo clarity: prior versions did not make the field-control function audible. Revised to sound each field state first, then let the same guide snap to the new state.
- Decision: demote from the curated v0.4 alpha path. Keep as a research comparison unless it later becomes musically legible without explanation.
